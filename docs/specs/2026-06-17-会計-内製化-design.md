# 設計書：会計（レジ・金種・伝票）の内製化

## 1. 背景と目的

hpb-calendar の 1cs POS 内製化の第2サブプロジェクト。
フェーズ1（予約管理の内製化）で予約レコードに `visitStatus`（予約→来店→未精算→精算済）軸と複数メニュー `menuItems[]` を導入済み。
本フェーズは、最後に1csへ依存している会計（レジ・金種・伝票）を内製化する。

- **予約レコード（`reservations-queue.json`）をライフサイクル全体の正**とし、会計確定時に会計情報を予約に書き込む。
- **`sales-log.json` を追記専用の売上台帳**とし、集計（`sales.js`）・領収書（`receipt.js`）が読む。
- 1cs への送信は行わない（1cs依存はゼロ）。SalonBoard 送信は予約の枠ブロック用に維持（本フェーズの対象外）。

### 決定事項（brainstorming 2026-06-17）

| 項目 | 決定 |
|---|---|
| 金種 | 会計時の**預かり/お釣り計算**（テンキー入力・お釣り自動計算）。レジ金の金種別在高カウントは閉店処理フェーズ。 |
| 会計起点 | **予約プレートから直接会計**（予約の menuItems を自動反映・visitStatus 連動） |
| 明細編集 | 会計時に**メニュー追加/削除＋物販（商品）追加**ができる |
| 割引 | **金額と % の両方**・会計全体に適用 |
| 物販マスタ | **admin で編集できる商品リスト**（`product-master.json` + CRUD） |
| データの持ち方 | **予約中心＋売上台帳**（予約=正、sales-log=追記専用） |

## 2. ゴール / 非ゴール

**ゴール**
- 予約プレートから起動する会計モーダルを admin に実装する
- 予約の menuItems を初期明細とし、メニュー追加/削除・物販追加・数量変更ができる
- 割引（金額/%）・支払方法・預かり/お釣りを扱い、会計を確定する
- 会計確定で予約に会計情報を書き、`sales-log.json` に複数明細の売上を追記する
- 伝票No（`YYYYMMDD-NNN`）を採番する
- 物販マスタ（`product-master.json`）を admin で CRUD できる
- 領収書（複数明細・預かり/お釣り・内税・伝票No）を発行する

**非ゴール（別サブプロジェクト）**
- レジ金の金種別在高カウント・レジ締め … 閉店処理フェーズ
- 集計ダッシュボードの拡充 … 集計フェーズ（本フェーズは `sales.js` を複数明細に追従させる最小改修のみ）
- 軽減税率（単一税率10%・税込で確定）
- 在庫管理（物販は名前・価格のみ。在庫数は持たない）
- SalonBoard/1cs への会計送信

## 3. データモデル

### 3.1 物販マスタ `product-master.json`（R2／新規）
```json
[
  { "id": "p001", "name": "シャンプー 250ml", "price": 2200, "active": true }
]
```
- `id`：`p` + 連番（採番は API 側）。`price`：税込。`active:false` は会計の選択肢から除外（履歴は保持）。

### 3.2 予約レコードに `checkout` を追加（`reservations-queue.json`）
会計確定時に予約レコードの直下へ追加し、`visitStatus` を `"paid"` にする。
```json
{
  "visitStatus": "paid",
  "checkout": {
    "slipNo": "20260617-001",
    "items": [
      { "kind": "service", "code": "T010001", "name": "【平日通常】リタッチ", "price": 2980, "qty": 1 },
      { "kind": "product", "id": "p001", "name": "シャンプー 250ml", "price": 2200, "qty": 1 }
    ],
    "discount": { "type": "amount", "value": 500 },
    "subtotal": 5180,
    "discountAmount": 500,
    "total": 4680,
    "taxIncluded": 425,
    "payment": "cash",
    "tendered": 5000,
    "change": 320,
    "paidAt": "2026-06-17T04:30:00.000Z"
  }
}
```
- `discount.type`：`"amount"`（円）または `"percent"`（%）。`value` は数値。
- `taxIncluded`：内消費税10% = `round(total - total / 1.1)`。
- 現金以外（card/qr）は `tendered`/`change` を `null` にできる。

### 3.3 売上台帳 `sales-log.json`（拡張）
確定時に1会計＝1レコード追記。複数明細に対応。
```json
{
  "reservationId": "uuid",
  "slipNo": "20260617-001",
  "date": "2026-06-17",
  "time": "12:30",
  "customer": "山田 花子",
  "phone": "090-1234-5678",
  "items": [ { "kind": "service", "name": "...", "price": 2980, "qty": 1 } ],
  "subtotal": 5180,
  "discount": { "type": "amount", "value": 500 },
  "discountAmount": 500,
  "total": 4680,
  "taxIncluded": 425,
  "payment": "cash",
  "tendered": 5000,
  "change": 320,
  "staff": "",
  "checkoutAt": "2026-06-17T04:30:00.000Z"
}
```
- **後方互換**：既存の旧レコードは単一 `menuName`/`finalPrice` を持つ。集計（`sales.js`）・領収書（`receipt.js`）は「`items[]` があればそれを使い、無ければ旧 `menuName`/`price`/`finalPrice` にフォールバック」する。

### 3.4 飛び込み（walk-in）
予約のない客は、会計開始時に `source:"walkin"` の予約レコードを `reservations-queue.json` に新規作成（`status:"completed"`＝SalonBoard送信対象外, `visitStatus:"arrived"`）してから会計フローに入る。SalonBoard へは送らない。

## 4. 計算ロジック `lib/checkout.js`（純粋関数・新規）

副作用なし・ストレージ非依存。会計の計算をすべてここに集約しテスト可能にする。

```
calcSubtotal(items)              → Σ (price × qty)
applyDiscount(subtotal, discount)→ { discountAmount, total }
                                   type==="amount": discountAmount = min(value, subtotal)
                                   type==="percent": discountAmount = round(subtotal × value / 100)
                                   total = max(subtotal - discountAmount, 0)   // 0 クランプ
calcTax(total)                   → round(total - total / 1.1)                  // 内税10%
calcChange(total, tendered)      → tendered - total （tendered < total は例外を投げる）
nextSlipNo(date, salesSameDay)   → `${YYYYMMDD}-${String(salesSameDay+1).padStart(3,"0")}`
computeCheckout({items, discount, payment, tendered})
                                 → { subtotal, discountAmount, total, taxIncluded, change }
                                   （現金以外は change=null・tendered検証スキップ）
```
- 採番の同時実行衝突は単一サロン運用のため許容（会計確定直前に当日 sales 件数 +1）。

## 5. API

### 5.1 `api/products.js`（新規・物販マスタ CRUD）
- `GET /api/products`：`product-master.json` を返す（`?activeOnly=1` で active のみ）。
- `POST /api/products`：`{ name, price }` で追加、または `{ id, name, price, active }` で更新。
- `DELETE /api/products?id=p001`：論理削除（`active:false`）。
- 認証：`Authorization: Bearer ${COMINGSOON_PASSWORD}`。

### 5.2 `api/checkout.js`（書き換え・予約ベース）
- Body：`{ reservationId, items[], products[], discount:{type,value}, payment, tendered }`
  - `items[]`：施術明細（reserve の menuItems を引き継ぎ＋当日追加）。`products[]`：物販明細。サーバ側で `kind` を付与し1本の `items[]` に統合。
- 処理：
  1. 認証・必須チェック（reservationId / 明細1件以上 / payment）
  2. `reservations-queue.json` から対象予約を取得（無ければ404／既に `visitStatus==="paid"` は409）
  3. `lib/checkout.computeCheckout` で計算（現金で `tendered < total` は400）
  4. `nextSlipNo` で伝票No採番（当日 sales 件数から）
  5. 予約に `checkout` を書き、`visitStatus="paid"` に更新 → `reservations-queue.json` 保存
  6. `sales-log.json` に売上レコード追記
  7. レスポンス `{ ok:true, slipNo, total, change, sale }`
- LINE通知・サンクスメール（email があれば）は既存挙動を維持。

### 5.3 来店遷移（reserved → arrived）
予約プレートの[来店]ボタン用。既存 `api/checkin.js` を予約 `visitStatus` 更新に寄せるか、`api/admin-action.js` に `setVisitStatus` を追加（実装計画で既存コードを確認して軽い方を選ぶ）。本フェーズの主眼は会計確定であり、ここは最小改修。

### 5.4 `api/receipt.js`（修正）
- `items[]` があれば複数明細行を描画（施術／物販）。無ければ旧 `menuName` 単一行（後方互換）。
- 追加表示：小計・割引・**内消費税**・**お預かり/お釣り**・**伝票No**。
- 受付番号は `slipNo` を優先（無ければ従来の visitId 切り出し）。

### 5.5 `api/sales.js`（修正）
- 集計の金額は `total`（無ければ旧 `finalPrice`）を使用。カテゴリ別はメニュー明細から集計（`items[].kind==="service"` のカテゴリ、物販は「物販」カテゴリ）。
- 「未会計（当日来店済み）」のソースを `reservations-queue.json` の `visitStatus==="arrived"` に切り替え（visits-log 依存を解消）。

## 6. UI（admin.html）

### 6.1 予約プレート
- 各予約に `visitStatus` 遷移ボタン：**[来店]**（reserved→arrived）／**[会計]**（会計モーダルを開く）。`paid` は「精算済」バッジ＋[領収書]リンク。

### 6.2 会計モーダル
1. 明細表：予約の `menuItems` を初期表示。各行に数量・小計、行削除。
2. 追加：**[メニュー追加]**（menu-master から選択）／**[物販追加]**（product-master から選択）。
3. 小計 → 割引（`[金額▼/％▼]` + 値入力）→ **合計**（リアルタイム再計算）。
4. 支払方法（現金/カード/QR）。現金時のみ **お預かり入力 → お釣り自動表示**。
5. **[会計確定]** → `POST /api/checkout` → 成功で領収書を新規タブ表示、予約プレートを再描画。

### 6.3 物販マスタ管理（設定セクション）
- 商品一覧（名前・価格・active）＋ 追加フォーム ＋ 行編集／無効化。`/api/products` を叩く。

## 7. データフロー

```
[予約プレート] [来店] → visitStatus: reserved→arrived
[予約プレート] [会計] → 会計モーダル
   → menu-master / product-master を選択して明細編集
   → POST /api/checkout
        → lib/checkout.computeCheckout（小計・割引・内税・お釣り）
        → nextSlipNo（当日 sales 件数+1）
        → reservations-queue.json 更新（checkout 情報 + visitStatus=paid）  ← 予約=正
        → sales-log.json 追記（複数明細）                                   ← 売上台帳
        → (email有) サンクスメール / LINE通知
   → 領収書 GET /api/receipt（複数明細・預かり/お釣り・内税・伝票No）
   → 集計 GET /api/sales（total ベース・未会計=visitStatus arrived）
```

## 8. エラーハンドリング
- 必須（reservationId・明細1件以上・payment）欠落 → 400
- 現金で `tendered < total`（お釣りマイナス）→ 400
- 予約が見つからない → 404
- 既に `visitStatus==="paid"`（二重会計）→ 409
- 割引が合計を超える → `total` を0クランプ（エラーにしない）
- R2 書込失敗 → 500（admin はトーストで通知しモーダルのデータを保持）
