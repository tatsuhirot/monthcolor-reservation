# HPB → 自社サイト（SalonBoard）自動登録フロー

> HPB（Hot Pepper Beauty）でお客様が予約 → メール通知 → 自社SalonBoardカレンダーに自動登録

```mermaid
flowchart TD
    A[📧 HPBから予約通知メール届く] --> B[メール本文を emails/*.txt に保存]
    B --> C[node import_from_email.js 実行]

    C --> D{salonboard-state.json\n存在する？}
    D -- なし --> E[ブラウザが開く\n手動ログイン]
    E --> F[セッション保存\nstate/salonboard-state.json]
    D -- あり --> G[セッション復元\n自動ログイン]
    F --> H
    G --> H

    H[メールファイルを順番に処理]

    H --> I[📧 メール本文を解析]
    I --> I1[予約番号 / 来店日時]
    I --> I2[氏名（漢字・カナ）]
    I --> I3[スタッフ名 → stylistId変換]
    I --> I4[クーポン名]

    I1 & I2 & I3 & I4 --> J

    J[SalonBoard 登録フォームを開く\n/CLP/bt/reserve/ext/extReserveRegist/]

    J --> K[フォームに自動入力]
    K --> K1[スタッフ・時間選択]
    K --> K2[クーポン選択]
    K --> K3[氏名入力]
    K --> K4[メモ欄に予約番号]

    K1 & K2 & K3 & K4 --> L[登録ボタンをクリック]

    L --> M{登録成功？}
    M -- ✅ 成功 --> N[次のメールへ]
    M -- ❌ 失敗 --> O[エラーログ出力]
    O --> N

    N --> P{まだメールがある？}
    P -- はい --> H
    P -- いいえ --> Q[✅ 完了\nX/Y件 登録成功]
```

## 関連ファイル

| ファイル | 役割 |
|---|---|
| `emails/*.txt` | HPBメール本文を保存する場所 |
| `import_from_email.js` | メール解析 → SalonBoard一括登録 |
| `parse_calendar.js` | カレンダーHTMLを解析して予約枠をJSON出力 |
| `register_reservation.js` | 単体で1件だけ登録する場合に使用 |
| `state/salonboard-state.json` | ログインセッション保存（自動生成） |
| `hpb-register.html` | 登録フォームの参考HTML |
| `hpb-reserve-calendar.html` | カレンダーの参考HTML |
